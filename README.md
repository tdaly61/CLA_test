# Welcome to the Mifos Initiative

At Mifos we have a Contributor License Agreement that must be signed prior to submitting PRs to our repos.

Why do we do this:

As an organisation we need to ensure contributors are following some simple rules and agree to abide by our code of conduct https://mifos.org/resources/community/code-of-conduct/ .  It is also essential for ensuring we maintain our licences and you understand your responsibilities.

Enforcement of our CLA:

Previously we relied on our PR reviewers checking that a CLA had been signed.  This was a manual step.  We have a growing number of contributors and we want to ensure our PR reviewers actually spend time reviewing the code so we are moving to an enforcement model that means the check is undertaken prior to the PR being raised.  We do this by locking down the rights to raise a PR to those who have signed the CLA.

Once you have signed our Contributor Licence Agreement at: https://mifos.org/about-us/financial-legal/mifos-contributor-agreement

Once it has been verified you will receive a request in Github to join our organisation as a member.

Once you have accepted this request you will be able to raise PRs.

You only need to sign our CLA once.  Unless you request to have it revoked it will remain in effect for all contributions under the Github ID you entered.  If you use more than one Github ID then you will need to sign the CLA for each Github ID you want to use.

This Repo is a Test repo.  Lets face it we cant always remember if we signed something.  You test if you have the ability to raise a PR against this repo.  If you don't have access it means you havent signed the CLA or you haven't accepted the invite to our organisation.

If this is the case please sign the CLA or reach out to us at dpi@mifos.org with your Github ID and we can investigate.
